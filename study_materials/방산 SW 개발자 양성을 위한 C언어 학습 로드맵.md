# 방산 SW 개발자 양성을 위한 C언어 학습 계획서

### 1. 제안 배경

* **방위산업의 특수성:** 무기 체계 및 전장 관리 시스템은 실시간성(Real-time)과 고신뢰성(High Reliability)을 최우선으로 요구함.
* **언어의 중요성:** 하드웨어 제어가 용이하고 실행 속도가 빠른 C언어는 방산 임베디드 시스템 개발의 표준 언어로 자리 잡고 있음.
* **필요성:** 비전공자 및 초심자가 방산 분야에 진입하기 위해서는 단순 문법 습득을 넘어, 메모리 구조 이해 및 하드웨어 제어 능력을 포함한 체계적인 학습 로드맵이 필요함.

### 2. 개발 목표

* **C언어 기초 확립:** 절차 지향 프로그래밍의 이해 및 문법 숙달.
* **메모리 제어 능력 배양:** 포인터 및 메모리 관리 기법을 통한 하드웨어 제어 기초 마련.
* **임베디드 사고방식 함양:** 제한된 자원 하에서의 효율적이고 안전한 코드 작성 능력 확보.

### 3. 국내외 현황

* **국내 현황:** LIG넥스원, 한화시스템, KAI 등 주요 방산 기업의 임베디드 소프트웨어는 대부분 C/C++ 기반으로 개발됨.
* **기술 현황:** 기존 레거시 시스템 유지보수 및 신규 무기 체계 개발 모두 C언어의 의존도가 매우 높으며, 무결성 검증을 위한 정적 분석 도구 활용이 보편화됨.

### 4. 핵심 기술

* **메모리 관리:** 포인터(Pointer)를 활용한 직접적인 메모리 접근 및 제어, 동적 메모리 할당/해제, 메모리 누수 방지.
* **비트 연산:** 레지스터 제어를 위한 비트 단위 데이터 조작(Bitwise Operation), 마스크 기법, 비트 필드 활용.
* **구조체 및 공용체 활용:** 통신 프로토콜 패킷 설계 및 데이터 구조화, 메모리 효율적 데이터 관리.
* **임베디드 C 특화 문법:** `volatile`, `static`, `const`, `extern` 등의 키워드를 활용한 최적화 방지 및 변수 범위 제어.
* **파일 입출력:** 시스템 로깅 및 데이터 영속성을 위한 파일 처리 기법.
* **에러 핸들링:** 방어적 프로그래밍 패턴 및 안전한 코드 작성 기법.
* **코드 품질 관리:** 정적 분석 도구를 활용한 코드 검증 및 MISRA C 규칙 준수.

### 5. 개발 환경

* **H/W:** PC(Windows/Linux), 임베디드 실습 보드(STM32 또는 Arduino 권장).
* **S/W:** Visual Studio Code, Visual Studio 2022.
* **Compiler:** GCC(GNU Compiler Collection), MSVC.
* **Tool:** 
  * Git(형상 관리)
  * 정적 분석 도구: Cppcheck, PC-lint(상용), 또는 GCC의 `-fanalyzer` 옵션
  * 메모리 디버거: Valgrind(Linux), AddressSanitizer, 또는 Visual Studio의 메모리 디버거

### 6. 개발 방법

* **집약적 학습:** 기초 문법부터 심화 메모리 관리, 실전 프로젝트까지 4주간 집중 학습.
* **이론 및 실습 병행:** 개념 학습 후 즉시 코드로 구현하여 동작 원리 파악.
* **디버깅 생활화:** IDE의 디버거를 활용하여 메모리 상태 및 변수 변화를 시각적으로 확인하며 학습.
* **실무 도구 활용:** 정적 분석 도구를 통한 코드 품질 검증 습관화.

* **포인터 기초:** 메모리 주소 개념, 주소 연산자(`&`)와 참조 연산자(`*`), 포인터 변수 선언 및 초기화.
* **포인터 심화:** 포인터와 배열의 관계, 포인터 연산(증감, 산술 연산), `void` 포인터, 이중 포인터, 함수 포인터 기초.
* **동적 메모리 관리:** 동적 메모리 할당(`malloc`, `calloc`, `realloc`), 메모리 해제(`free`), 메모리 누수(Memory Leak) 방지 기법, `NULL` 포인터 체크 습관화.
* **구조체와 공용체:** 구조체(Struct) 정의 및 활용, 구조체 포인터, 공용체(Union)의 메모리 효율적 활용, 구조체 배열.
* **파일 입출력:** 파일 열기/닫기(`fopen`, `fclose`), 텍스트/바이너리 파일 읽기/쓰기(`fread`, `fwrite`, `fscanf`, `fprintf`), 파일 포인터 위치 제어(`fseek`, `ftell`).
* **실습:** 동적 배열 구현, 구조체 기반 데이터베이스 프로그램, 파일 기반 로깅 시스템, 메모리 디버깅 도구 활용.

#### 3주차: 방산/임베디드 심화 및 자료구조 (2026.01.03 ~ 2026.01.09)

* **비트 연산:** 비트 연산자(AND `&`, OR `|`, XOR `^`, NOT `~`, SHIFT `<<`, `>>`), 마스크(Mask) 기법, 비트 필드(Bit Field), 레지스터 제어 시뮬레이션.
* **전처리기 및 모듈화:** 전처리기 지시자(`#define`, `#include`, `#ifdef`, `#ifndef`, `#pragma`), 매크로 함수, 조건부 컴파일, 헤더 파일 가드(`#ifndef`), 분할 컴파일 및 링킹.
* **임베디드 필수 키워드:** `volatile`(하드웨어 레지스터 접근), `static`(변수/함수 범위 제어), `extern`(외부 변수 선언), `const`(상수성 보장), `register`(레지스터 변수), 저장소 클래스 이해.
* **함수 포인터 및 콜백:** 함수 포인터 선언 및 활용, 콜백 함수 구현, 이벤트 핸들러 패턴, 함수 포인터 배열.
* **에러 핸들링:** 반환값 기반 에러 처리, 에러 코드 정의, `errno` 활용, 방어적 프로그래밍(Defensive Programming) 패턴.
* **MISRA C 가이드라인:** MISRA C:2012 핵심 규칙 학습(포인터 사용 제한, 타입 변환 규칙, 매크로 사용 규칙), 정적 분석 도구를 통한 규칙 위반 검출.
* **자료구조 기초:** 연결 리스트(Linked List) 구현, 스택(Stack) 및 큐(Queue) 구현, 배열 기반 vs 포인터 기반 자료구조 비교.
* **실습:** 비트 마스킹을 활용한 레지스터 시뮬레이터, 모듈화된 프로젝트 구조 설계, 함수 포인터 기반 이벤트 시스템, MISRA C 규칙 준수 코드 작성.

  * **3주차:** 임베디드 심화 지식 평가 (비트 연산, MISRA C 규칙 준수 코드 작성)
  * **4주차:** 실전 프로젝트 완성도 평가 (기능 구현, 코드 품질, 문서화)

* **메모리 검증:** 동적 할당된 메모리의 해제 여부 확인(Memory Leak 점검), 정적 분석 도구를 통한 코드 품질 검증.

### 10. 예상 문제점

* **집약적 학습으로 인한 부담:** 4주간의 집약 과정에서 학습량이 많아 피로도가 높을 수 있음.
  * *해결 방안:* 일일 학습 목표 설정, 주기적인 복습 시간 확보, 실습 위주 학습으로 이해도 향상.

* **포인터 개념 습득의 어려움:** 포인터와 메모리 구조 이해 과정에서 학습 난이도가 급격히 상승할 수 있음.
  * *해결 방안:* 메모리 구조 그림 그리기(Memory Map) 및 디버거를 통한 주소 값 확인 훈련 병행, 단계별 예제 코드 분석.

* **하드웨어 제약 사항 이해 부족:** PC 환경과 임베디드 환경의 차이(메모리 크기 등) 인식 부족.
  * *해결 방안:* 자료형의 크기(`sizeof`) 확인 습관화 및 임베디드 제약 사항에 대한 지속적인 이론 학습, 시뮬레이션 환경 활용.

* **정적 분석 도구 활용의 어려움:** 초기 설정 및 결과 해석에 시간이 소요될 수 있음.
  * *해결 방안:* 무료 도구(Cppcheck)부터 시작하여 점진적으로 고급 도구 활용, 도구 매뉴얼 및 튜토리얼 참고.

### 11. 차별적 우수성

* **방산 특화 커리큘럼:** 단순 알고리즘 풀이가 아닌, 비트 연산 및 메모리 제어 등 하드웨어 밀접 기술에 집중.
* **신뢰성 중심 학습:** MISRA C 가이드라인 맛보기를 통해 "돌아가는 코드"가 아닌 "안전한 코드" 작성을 지향.
* **표준 준수:** C99/C11 등 C언어 표준 규격을 준수하여 이식성 높은 코드 작성 능력 배양.

### 12. 기대 효과

* **직무 적합성 확보:** 방산 무기 체계 SW 개발 직무 기술서(JD)에 부합하는 핵심 역량을 4주간 집약적으로 확보.
* **실무 도구 활용 능력:** 정적 분석 도구 및 메모리 디버거 활용 능력을 통해 실무 환경에 즉시 적응 가능.
* **확장성:** C언어의 깊은 이해를 바탕으로 향후 C++, RTOS(실시간 운영체제), 리눅스 커널 등으로 기술 영역 확장 용이.
* **문제 해결력 향상:** 컴퓨터 구조와 메모리 동작 원리를 이해함으로써 시스템 레벨의 디버깅 능력 향상.
